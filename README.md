# SystemIo - Laboratory Management API

Сервис для расчета стоимости лабораторных исследований с учетом налоговых ставок разных стран и системы скидочных купонов.

## 🚀 Технологический стек
* **Java 17** 
* **Spring Boot 3.4.1**
* **Maven**
* **PostgreSQL** (СУБД)
* **Liquibase** (Миграции базы данных)
* **OpenAPI / Swagger** (Документация API)
* **Lombok** (Генерация кода)

---

## 🛠 Запуск проекта

### 1. Требования
* Установленный **Docker** и **Docker Compose**.
* **Java 17** (если запускаете без Docker).

### 2. Быстрый старт (Docker)
Самый простой способ поднять приложение вместе с базой данных:

```bash
docker-compose up -d
```


### 📖 Документация API (Swagger)
После запуска приложения интерактивная документация доступна по адресу:
👉 http://localhost:8080/swagger-ui/index.html

Здесь вы можете протестировать основные эндпоинты:

POST /calculate-price — расчет финальной стоимости.

POST /purchase — проведение платежа через выбранный процессор (Paypal/Stripe).


### 🧪 Тестирование
Проект покрыт Unit-тестами для сервисов и MockMvc-тестами для контроллеров.
Запуск тестов:

```bash
mvn test
```


### База данных и миграции
Проект использует PostgreSQL в связке с Liquibase для управления миграциями. Это гарантирует, что структура таблиц будет идентична на всех машинах разработчиков и в Docker-контейнере.

Схема данных
Основные сущности системы:

Product: Хранит информацию о доступных товарах и их базовой цене (без учета налогов).

Coupon: Содержит промокоды и процент скидки (например, P10 -> 10%).


### 🛡 Валидация данных (Validation Logic)

Валидация в проекте реализована на двух уровнях: DTO (Bean Validation) и Custom Annotations.

 Кастомная валидация налоговых номеров
 Для проверки формата taxNumber создана аннотация @TaxNumber. 
 Логика проверки инкапсулирована в TaxNumberValidator:

Германия (DE): DE + 9 цифр (например, DE123456789).

Италия (IT): IT + 11 цифр.

Греция (GR): GR + 9 цифр.

Франция (FR): FR + 2 буквы + 9 цифр.
Структура ошибки 400 (Bad Request):
Если налоговый номер не соответствует маске страны или формат купона неверен, API вернет структурированный JSON:

```JSON
{
  "errorCode": "VALIDATION_ERROR",
  "message": "Ошибка валидации входящих параметров",
  "timestamp": "2026-03-22T21:47:32",
  "errors": [
    {
      "field": "taxNumber",
      "rejectedValue": "DE123",
      "message": "Налоговый номер должен состоять из DE и 9 цифр"
    }
  ]
}
```
